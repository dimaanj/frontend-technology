# Последовательное выполнение промисов — саммари

## Условие

Написать функцию `sequentialPromises(items, asyncFn)`:

- **Вход:** массив `items` и асинхронная функция `asyncFn(item)`.
- **Поведение:** вызывать `asyncFn` для каждого элемента **по очереди** — следующий вызов только после завершения предыдущего.
- **Выход:** промис, резолвящийся в **массив результатов** в порядке выполнения.
- **Ошибка:** если любой вызов падает — выполнение останавливается, ошибка пробрасывается.
- **Пустой массив** → `[]`.

## Пример

```js
const items = ['item1', 'item2', 'item3'];
const postData = async (item) => {
  await delay(100);
  return `posted: ${item}`;
};
const results = await sequentialPromises(items, postData);
// results = ['posted: item1', 'posted: item2', 'posted: item3']
// Время ~300ms (100ms × 3)
```

## Способы решения

1. **Цикл `for` / `for...of` + `await`** — на каждой итерации `await asyncFn(item)`, результат пушить в массив. Самый прямой и читаемый вариант.
2. **`reduce`** — цепочка промисов: `items.reduce((prev, item) => prev.then(() => asyncFn(item)).then(r => res.push(r)), Promise.resolve())`. Порядок гарантирован; результат собирается через side effect.
3. **Рекурсия** — обработать первый элемент, рекурсивно вызвать себя для `rest`, вернуть `[head, ...tail]`. При большом `items.length` возможен переполнение стека.

## Сложность

- **Время:** O(n × T), где n — длина массива, T — среднее время одного вызова `asyncFn`. Вызовы строго последовательны.
- **Память:** O(n) — массив результатов; при рекурсии дополнительно глубина стека вызовов O(n).

## Краевые случаи

- Пустой массив → вернуть `[]`, `asyncFn` не вызывать.
- Один элемент → один вызов, массив из одного результата.
- Ошибка на первом элементе → сразу throw, остальные не запускаются.
- Ошибка в середине → обработанные результаты уже получены, выполнение прерывается, ошибка пробрасывается.

## Связь с async iterator

Для **обычного массива** async iterator (`for await...of`) не даёт последовательности — массив итерируется синхронно. Последовательность достигается за счёт `await` в теле цикла. Async iterator нужен, когда источник данных сам асинхронный (стримы, пагинация, очереди) — см. `async-iterator-examples.js`.
