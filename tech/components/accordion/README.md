# Accordion — что сделано

Краткое описание реализации компонента: базовые требования и доступность (a11y).

---

## Базовые требования

### Несколько секций (минимум 3)

- Данные заданы в массиве `accordionData` (5 пунктов).
- Рендер через `accordionData.map()` — каждый элемент даёт один блок «заголовок + контент».

### Клик по заголовку раскрывает/скрывает содержимое

- Заголовок — нативный `<button>` с `onClick={() => toggleAccordion(item.id)}`.
- В `toggleAccordion(id)`:
  - если открыта та же секция (`openId === id`) — закрываем: `setOpenId(null)`;
  - иначе открываем выбранную: `setOpenId(id)`.

### Плавная анимация при открытии/закрытии

- В CSS контент обёрнут в контейнер с **CSS Grid**: `grid-template-rows: 0fr` (закрыто) и `1fr` (открыто).
- Внутри — блок с `overflow: hidden` (`.accordion-content-inner`), высота анимируется через `transition: grid-template-rows 0.3s ease`.
- Так анимация не зависит от фиксированной `max-height` и корректно работает при любой длине текста.

### Только одна секция открыта одновременно

- Состояние хранится в одном значении: `openId` (id открытой секции или `null`).
- При открытии новой секции просто вызывается `setOpenId(id)` — предыдущая автоматически считается закрытой.

### Визуальная индикация открытой секции (стрелка)

- У каждого пункта на кнопке есть иконка-стрелка (`.accordion-icon`).
- Для открытого пункта на контейнер вешается класс `open`; в CSS: `.accordion-item.open .accordion-icon { transform: rotate(180deg); }`.
- Стрелка плавно поворачивается за счёт `transition: transform 0.3s ease`.

### Возможность закрыть текущую открытую секцию

- Реализовано в `toggleAccordion`: при клике по уже открытой секции вызывается `setOpenId(null)` — секция закрывается.

### Адаптивный дизайн

- В `@media (max-width: 480px)` уменьшены отступы у `body` и у кнопки-заголовка (padding, font-size), чтобы на узких экранах блок выглядел аккуратно.

## Доступность (a11y)

### Семантическая вёрстка

- **Секция и заголовок группы:** блок аккордеона обёрнут в `<section aria-labelledby="accordion-title">`. Внутри — скрытый от визуала, но доступный скринридерам заголовок `<h2 id="accordion-title">` («Frequently asked questions»). Так скринридер объявляет назначение блока.
- **Иерархия заголовков:** каждая секция — `<h3>` с кнопкой внутри. Это даёт структуру «вопрос — ответ» и логичный уровень заголовков (h1 страницы → h2 группа FAQ → h3 отдельный вопрос).
- **Кнопка:** используется нативный `<button type="button">`, а не `div` с ролью — даёт встроенную клавиатурную активацию (Enter/Space) и корректную семантику без лишнего JS.
- **Скрытие декоративной иконки:** у стрелки стоит `aria-hidden="true"`, чтобы её не озвучивали отдельно.

### Навигация с клавиатуры

- **Tab / Shift+Tab:** фокус перемещается только по кнопкам заголовков (и далее по странице) — порядок табуляции естественный.
- **Enter / Space:** на кнопке активируют открытие/закрытие секции (поведение кнопки по умолчанию).
- **Стрелки и Home/End:** добавлен обработчик `onHeaderKeyDown`:
  - **ArrowDown** — фокус на следующую кнопку (с последней — на первую).
  - **ArrowUp** — на предыдущую (с первой — на последнюю).
  - **Home** — на первую кнопку.
  - **End** — на последнюю.
- Переход между кнопками делается через `buttonRefs.current[index].focus()`; для стрелок/Home/End вызывается `event.preventDefault()`, чтобы не скроллить страницу.

### ARIA-атрибуты

- **Кнопка заголовка:**
  - `aria-expanded={isOpen}` — скринридер объявляет «развёрнуто» / «свёрнуто».
  - `aria-controls={panelId}` — связь с панелью контента по `id`.
  - У кнопки уникальный `id={buttonId}` для обратной связи с панелью.
- **Панель контента:**
  - `id={panelId}` — тот же id, что в `aria-controls`.
  - `role="region"` — явная роль раскрываемой области.
  - `aria-labelledby={buttonId}` — объявление связывает панель с заголовком-кнопкой.
  - `aria-hidden={!isOpen}` — скрытая панель не читается как активный контент.
- **Контейнер аккордеона:** у внутреннего `div` с классом `.accordion` стоит `role="presentation"`, чтобы не вводить лишнюю структуру для скринридеров; смысловая нагрузка — у `section` и кнопок/панелей.

### aria-live

- Добавлена скрытая от визуала, но озвучиваемая область: `<p className="visually-hidden" aria-live="polite" aria-atomic="true">`. В ней выводится текст `announcement`.
- При открытии секции: «[название вопроса] expanded», при закрытии: «[название вопроса] collapsed».
- `aria-live="polite"` — объявление не перебивает текущую речь. `aria-atomic="true"` — читается весь текст сообщения целиком.
- Класс `.visually-hidden` скрывает блок визуально, но оставляет его в дереве доступности для скринридеров.

### Фокус

- **Видимость фокуса:** для кнопки задано `.accordion-header:focus-visible` (синий outline). Фокус виден при навигации с клавиатуры и не показывается при клике мышью (если браузер не рисует focus после клика).
- **Где остаётся фокус:** при открытии и закрытии секции фокус остаётся на кнопке заголовка. Панель контента не получает фокус и не входит в порядок табуляции — так проще и предсказуемее для клавиатуры и скринридеров.
- **Ловушка фокуса (focus trap):** не используется — для аккордеона это не требуется; фокус может свободно уходить на другие элементы страницы.

### Вспомогательное: класс «visually-hidden»

- В CSS описан класс `.visually-hidden`: элемент остаётся в DOM и доступен скринридерам, но визуально скрыт (уменьшенная область, обрезка, overflow). Используется для заголовка секции и для live-области с объявлениями.

## Итог

- Все пункты из базового ТЗ выполнены: несколько секций, клик по заголовку, плавная анимация (grid), одна открытая секция, стрелка, закрытие по повторному клику, адаптив.
- Для a11y добавлены: семантика (section, h2, h3, button), полная клавиатурная навигация (Tab, Enter/Space, стрелки, Home/End), ARIA (expanded, controls, region, labelledby, hidden), объявления через aria-live и явное отображение фокуса.
