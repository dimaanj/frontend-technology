# 78. Subsets — итоги

## Решение

Итеративный подход: начинаем с одного подмножества `[[]]`. На каждом шаге берём текущий элемент `nums[i]`, для каждого уже построенного подмножества в `res` создаём новое — старый поднабор плюс этот элемент, и добавляем все такие новые подмножества в `res`.

## Сложность

- **По времени:** O(n · 2^n). Циклов по `i` — n; на i-м шаге в `res` уже 2^i подмножеств, для каждого делаем копию O(i) и конкатенацию. Суммарно количество копируемых элементов порядка n · 2^(n−1), итог O(n · 2^n).
- **По памяти:** O(n · 2^n) — хранение результата (2^n подмножеств, в сумме порядка n · 2^(n−1) элементов). Дополнительно массив `arr` на каждом шаге — O(2^n) элементов в пике; без учёта выходного массива дополнительная память O(2^n).

## Альтернативы

- **Битовые маски:** перебрать числа от 0 до 2^n − 1; каждый бит — брать или не брать элемент. Время O(n · 2^n), идея та же, код без явного «размножения» подмножеств.
- **Рекурсия/бэктрэкинг:** на каждом шаге «взять» или «не взять» элемент, в листьях пушить текущий путь в ответ. Та же асимптотика, другой стиль.

## Краевые случаи

- `nums = []` — по условию не бывает (1 ≤ nums.length ≤ 10).
- `nums = [0]` — ответ `[[], [0]]`.
- Уникальность по условию гарантирована — дубликаты подмножеств не возникают.
