# 46. Permutations — итоги

## Решение

Бэктрэкинг с обменом (swap): на шаге `depth` фиксируем префикс `nums[0..depth-1]`, в позицию `depth` по очереди ставим каждый из элементов `nums[depth..n-1]` (swap), рекурсивно строим перестановки от `depth+1`, после рекурсии откатываем swap. В листьях (`depth === nums.length`) в ответ кладём копию текущего массива.

В файле также есть альтернатива `permute1`: рекурсия «выбери первый элемент» — для каждого элемента как первого строятся перестановки остатка и он дописывается в начало. В экспорт не входит.

## Сложность

- **По времени:** O(n · n!). Всего n! перестановок, каждая в листе копируется за O(n), итого O(n · n!). Рекурсивных вызовов порядка n! (каждый лист — одна перестановка).
- **По памяти:** O(n) дополнительной памяти (глубина стека рекурсии и один массив на месте). Результат O(n · n!) — не считается в «дополнительной» при оценке задачи.

## Альтернативы

- **Swap-based (текущий permute):** минимум копий в процессе — копия только в листьях; один массив переставляется на месте.
- **«Первый элемент + рекурсия на rest» (permute1):** на каждом шаге создаётся новый массив `rest` и много промежуточных массивов; асимптотика по времени та же O(n · n!), по памяти выше из-за срезов и конкатенаций.
- **Итеративный (например, алгоритм Хипа):** генерация следующей перестановки за O(n), всего n! раз — та же O(n · n!) по времени, O(1) доп. памяти без учёта ответа.

## Краевые случаи

- Один элемент: `nums = [1]` → `[[1]]`.
- Два элемента: `[1, 2]` → `[[1, 2], [2, 1]]`.
- По условию LeetCode: массив из различных чисел, длина 1..10 — пустой ввод не рассматривается.
