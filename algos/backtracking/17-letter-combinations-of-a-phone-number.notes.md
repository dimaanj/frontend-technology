# 17. Letter Combinations of a Phone Number — итоги

## Решение

Бэктрэкинг по позиции в `digits`: на каждом шаге берётся цифра `digits[pos]`, по маппингу — строка букв; для каждой буквы рекурсивный вызов `backtrack(pos + 1, word + ch)`. Когда `word.length === digits.length`, строка добавляется в результат и выполняется `return` (важно не продолжать цикл по буквам, иначе обращение к `digits[pos]` при `pos === digits.length` даёт `undefined` и падение).

## Сложность

- **По времени:** O(4^n · n) в худшем случае. Комбинаций до 4^n (каждая цифра до 4 букв, n = digits.length); каждая строка длины n. С учётом конкатенации `word + ch` — до O(n) на одну строку, итог O(4^n · n).
- **По памяти:** O(n) — глубина рекурсии и строка `word` на стеке. Выходной массив 4^n строк по n символов — O(4^n · n), но обычно не учитывают в «дополнительной» памяти.

## Альтернативы

- **Итеративный BFS с очередью:** изначально в очереди одна пустая строка; на каждом «уровне» для каждой строки в очереди дописываем по очереди все буквы следующей цифры и кладём новые строки в очередь. Та же асимптотика по времени и памяти.
- **Итеративный с массивом результатов:** хранить массив текущих строк, для следующей цифры размножить каждую строку на все буквы этой цифры (как в BFS, но без явной очереди).

## Краевые случаи

- `digits = ""` → `[]` (в коде ранний возврат).
- Одна цифра (например `"2"`) → `["a","b","c"]`.
- Максимум по условию: 4 цифры — до 4^4 = 256 комбинаций.
