# 392. Is Subsequence — итоги

## Условие

Даны строки `s` и `t`. Вернуть `true`, если `s` является подпоследовательностью `t`, иначе `false`. Подпоследовательность — строка, полученная удалением части символов без изменения порядка оставшихся (например, `"ace"` — подпоследовательность `"abcde"`, `"aec"` — нет).

## Решение

Два указателя: `idx` по `s`, проход по `t` слева направо. При совпадении `s[idx] === t[i]` сдвигаем `idx`. Если дошли до последнего символа `s` и он совпал — возвращаем `true`. Пустая `s` — подпоследовательность любой строки.

## Сложность

- **Время:** O(|t|) — один проход по `t`.
- **Память:** O(1) — только счётчик.

## Краевые случаи (покрыты тестами)

- Пустая `s` → `true`.
- Пустая `t` при непустой `s` → `false`.
- Один символ в `s`: есть в `t` / нет в `t`.
- `s` длиннее `t` → `false`.
- Сохранение порядка: `"ace"` в `"abcde"` — да, `"aec"` — нет.
- Повторяющиеся буквы в `t`: `"aaa"` в `"aaab"`, `"abc"` в `"abbc"`.
- Все символы `s` должны быть в `t` в нужном порядке: `"abc"` не подпоследовательность `"ac"`.

## Альтернативы

- Рекурсия/перебор — избыточно, хуже по времени.
- Бинарный поиск по позициям символов в `t` (предподсчёт индексов для каждой буквы) — выгоден при множестве запросов «is s subsequence of t» для одного и того же `t` (LeetCode 792).
