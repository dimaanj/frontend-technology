# 792. Number of Matching Subsequences — итоги разбора

Задача: даны строка `s` и массив строк `words`. Вернуть количество слов из `words`, являющихся подпоследовательностью `s`.

Решение: один проход по `s`. Слова сгруппированы в Map по **следующему ожидаемому символу** (ключ — буква, значение — список пар `[word, index]`). В начале все слова кладутся в bucket по первой букве. Для каждого символа `c` в `s` берётся bucket по ключу `c`, для каждой пары `[w, index]`: если `index + 1 === w.length` — слово посчитано; иначе пара `[w, index + 1]` добавляется в bucket по символу `w[index + 1]`.

---

## 1. Сложность

- **Время:** O(|s| + Σ|words[i]|).  
  Один проход по `s`; каждый символ каждого слова обрабатывается ровно один раз (при переносе в следующий bucket или при подсчёте). В ограничениях (|s| ≤ 5·10⁴, слов до 5000, длина слова до 50) — порядка 3·10⁵ операций.

- **Память:** O(|words| + сумма длин слов).  
  Map с не более чем 26 ключами; в списках хранятся пары `[word, index]` — одновременно «в полёте» не больше одного состояния на слово. Плюс ссылки на сами строки слов.

---

## 2. Альтернативы

- **Текущий подход (bucket по следующему символу):** один проход по `s`, слова перераспределяются по bucket'ам. Хорошая константа, линейная суммарная работа по символам.

- **Наивный:** для каждого слова проверить «является ли подпоследовательностью» (два указателя по `s` и по слову). Время O(W · (|s| + max |word|)), где W = words.length. При W = 5000 и |s| = 50 000 даёт до сотен миллионов сравнений — в JS может быть на грани по времени.

- **Бинарный поиск по вхождениям:** предподсчитать для каждой буквы отсортированный список индексов в `s`. Для каждого слова для каждой буквы — бинарный поиск следующего вхождения после текущей позиции. Время O(|s| + W · max |word| · log |s|). Выгодно при большом |s| и относительно коротких словах; по константе и реализации сложнее текущего решения.

---

## 3. Оптимизация

- Повторное использование одной строки `s` и группировка по следующему символу уже дают линейную по сумме длин сложность без лишних проходов по `s`.
- Дубликаты в `words` учитываются корректно: каждое вхождение слова кладётся в bucket отдельно и считается отдельно.
- При желании можно заменить Map на массив из 26 элементов (индекс по коду буквы) — чуть меньше накладных расходов, асимптотика та же.

---

## 4. Краевые случаи

- **Пустой `words`:** цикл по `s` не находит слов в bucket'ах, возвращается 0.
- **Слово длиннее `s`:** не может быть подпоследовательностью; индекс в слове не дойдёт до конца, слово не попадёт в счётчик.
- **Одинаковые буквы в `s`:** при повторном появлении буквы (например, `s = "aaa"`) слова из соответствующего bucket'а обрабатываются при каждом таком символе — порядок сохраняется, подсчёт корректен.
- **Один символ в слове:** при совпадении с символом в `s` на следующем шаге `newIndex === w.length`, слово сразу попадает в счётчик.
