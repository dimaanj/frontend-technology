# 340. Longest Substring with At Most K Distinct Characters — итоги

## Сложность

- **По времени:** O(n). Строка проходится указателями `l` и `r`; каждый символ добавляется в окно (правый шаг) и удаляется из окна (левый шаг) не более одного раза. Операции с Map — O(1) в среднем.
- **По памяти:** O(k). В `Map` одновременно хранится не более k+1 ключей (окно сжимается при `map.size > k`). При фиксированном алфавите можно уточнить как O(min(k, |Σ|)).

## Альтернативы

- **Sliding window + Map (текущее решение):** удобно считать количество различных символов через `map.size`, сжатие слева при превышении k.
- Перебор всех подстрок O(n²) с подсчётом distinct за O(n) или за O(1) при фиксированном алфавите — итого O(n²) или O(n²·|Σ|), хуже по времени.
- Вариант с двумя указателями и явным счётчиком distinct вместо `map.size` — та же асимптотика, чуть другой код.

## Оптимизация

- Критично при сжатии окна удалять ключ из Map при счётчике 1 (`map.delete`), иначе ключ с нулём остаётся, `map.size` не уменьшается и цикл не завершается.
- Дополнительных оптимизаций по асимптотике не требуется.

## Краевые случаи

- Пустая строка → 0.
- k = 0 → 0 (нет валидного окна).
- k ≥ число различных символов в строке → ответ равен длине строки.
- Один символ, k ≥ 1 → 1.
- Все символы одинаковые, k = 1 → длина строки.
