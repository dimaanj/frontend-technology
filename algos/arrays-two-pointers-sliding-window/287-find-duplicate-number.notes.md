# 287. Find the Duplicate Number — итоги

## Сложность

- **По времени:** O(n). Первая фаза: указатели встречаются за O(n). Вторая фаза: оба идут по одному шагу до встречи — тоже O(n).
- **По памяти:** O(1). Только две переменные (slow, fast).

## Решение

Алгоритм Флойда поиска входа в цикл. Массив трактуется как граф: из индекса `i` ведём ребро в `nums[i]`. Из-за дубликата в графе есть цикл; индекс входа в цикл совпадает с дубликатом (значением, которое повторяется).

1. **Фаза 1:** slow = 0, fast = 0. Do-while: slow = nums[slow], fast = nums[nums[fast]], пока не совпадут. Старт с индекса 0 нужен, чтобы была «хвост» и фаза 2 корректно находила вход в цикл.
2. **Фаза 2:** fast = 0. Пока slow ≠ fast: slow = nums[slow], fast = nums[fast] (оба по одному шагу). Точка встречи — индекс входа в цикл, он же дубликат; возвращаем его.

Важно: во второй фазе оба указателя двигаются на один шаг за итерацию. Вариант со стартом от nums[0] без 0 может зацикливать второй цикл (например, для [1,2,1]).

## Альтернативы

- **Хэш-множество:** один проход, запоминаем увиденные числа, первое повторное — ответ. O(n) время, O(n) память.
- **Бинарный поиск по ответу:** бинарный поиск по значению x из [1, n]; считаем, сколько элементов ≤ x. Если больше x — дубликат ≤ x. O(n log n) время, O(1) память.
- **Сортировка:** O(n log n) время, O(1) или O(log n) в зависимости от реализации; по условию массив менять нельзя.

## Краевые случаи (покрыты тестами)

- Минимальный массив [1,1].
- Дубликат в начале, в конце, все элементы одинаковые.
- Примеры из условия LeetCode.
