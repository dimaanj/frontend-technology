# Merge Sort — саммари

## Идея

**Divide and conquer:** массив рекурсивно делим пополам, сортируем каждую половину (до базового случая — один элемент), затем **сливаем** две отсортированные половины в один отсортированный массив.

## Алгоритм

1. **Базовый случай:** если `lo >= hi` (отрезок длины 0 или 1) — уже «отсортирован», выходим.
2. **Делим:** `mid = (lo + hi) >> 1`.
3. **Рекурсия:** `mergeSort(arr, lo, mid)` и `mergeSort(arr, mid + 1, hi)`.
4. **Слияние (merge):** два указателя по левой и правой половине, по одному элементу выбираем меньший и пишем во временный массив (или обратно в `arr` при in-place варианте).

## Слияние (merge)

- Два отсортированных отрезка: `[lo, mid]` и `[mid+1, hi]`.
- Указатели `i` (левый), `j` (правый). Сравниваем `arr[i]` и `arr[j]`, меньший записываем в результат, сдвигаем соответствующий указатель.
- Остаток одной из половин дописываем в конец.

## Сложность

|          | Время      | Память        |
|----------|------------|---------------|
| Среднее  | O(n log n) | O(n) — буфер  |
| Худшее   | O(n log n) | O(n)          |
| Лучшее   | O(n log n) | O(n)          |

- Высота дерева рекурсии: log n. На каждом уровне обрабатываем в сумме n элементов → время O(n log n).
- Доп. память: массив длины n для слияния (или O(n) на стек при рекурсии + буфер).

## Особенности

- **Стабильность:** да, если при слиянии при равенстве брать элемент из левой половины.
- **In-place:** классический merge не in-place; есть варианты (например, in-place merge) сложнее.
- **Хорош для:** связных списков, внешней сортировки (данные не влезают в память), когда нужна стабильность и предсказуемое O(n log n).

## Edge cases

- Пустой массив или один элемент — сразу возврат.
- Два элемента — один вызов merge после деления.
